# 大麦网抢票工具 (DamaiGrabber) 设计文档

## 概述

一个桌面 GUI 工具，帮助用户在大麦网开票瞬间自动完成"点击购买"和"确认订单"两个操作。用户在自己的浏览器中提前完成登录、选演出、选场次/票价/观演人等准备工作，工具通过 CDP 协议连接到用户浏览器，在开票时间精准执行抢票动作。

## 核心原则

- **最小介入**：工具只做两件事 — 点击"立即购买"和点击"提交订单"，其余操作由用户在浏览器中手动完成
- **精准触发**：通过 NTP 校时实现毫秒级定时精度
- **低检测风险**：Attach 到用户真实浏览器 + stealth 注入，最大程度避免被风控拦截

## 技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 语言 | Python 3.10+ | 生态丰富，开发效率高 |
| GUI | PyQt6 | 成熟的桌面 GUI 框架 |
| 浏览器自动化 | Playwright (async) | 通过 CDP 协议连接用户浏览器 |
| 反检测 | playwright-stealth | 隐藏自动化特征 |
| 时间同步 | ntplib | NTP 协议校时 |
| HTTP 客户端 | httpx | 备用，NTP 请求等场景 |
| 配置存储 | JSON | 轻量，用户可手动编辑 |

## 使用流程

```
1. 用户启动工具 → 点击"启动浏览器"
   (工具以 --remote-debugging-port=9222 启动 Chrome)

2. 用户在 Chrome 中完成准备工作：
   - 登录大麦网
   - 进入目标演出页面
   - 选择场次、票价档位、观演人
   - 停留在演出详情页（"立即购买"按钮所在页面）

3. 用户回到工具：
   - 设置开票时间（日期 + 时间，精确到秒）
   - 点击"开始抢票"

4. 工具执行：
   - NTP 校时，计算时间偏移
   - 精准倒计时
   - 开票瞬间：检测"立即购买"按钮 → 点击
   - 等待页面跳转 → 检测"提交订单"按钮 → 点击
   - 日志面板实时显示每一步的状态和耗时

5. 用户在浏览器中手动完成支付
```

## 模块详细设计

### 1. Playwright Manager (`core/browser.py`)

**职责**：管理与用户浏览器的连接。

**功能**：
- 以远程调试模式启动 Chrome（`--remote-debugging-port=9222`）
- 通过 `playwright.chromium.connect_over_cdp()` 连接到已运行的浏览器
- 注入 playwright-stealth 脚本
- 获取用户当前打开的大麦网页面 Tab（通过 URL 匹配）
- 提供页面连接状态查询

**关键接口**：
- `launch_browser()` → 启动带调试端口的 Chrome
- `connect()` → CDP 连接到浏览器
- `get_damai_page()` → 获取大麦网页面对象
- `is_connected()` → 连接状态

### 2. NTP 校时器 (`core/timer.py`)

**职责**：精准时间同步和定时触发。

**功能**：
- 连接 NTP 服务器（`ntp.aliyun.com`、`ntp.tencent.com` 等，多源容错）
- 计算本地时钟与 NTP 时间的偏移量（offset）
- 获取校正后的当前精准时间
- 提供 `wait_until(target_time)` 方法，精准等待到目标时间
  - 距离目标 >1s 时用 sleep 粗等
  - 距离目标 <1s 时用忙等待（busy-wait）实现毫秒级精度

**关键接口**：
- `sync()` → 执行 NTP 校时，返回偏移量
- `now()` → 获取校正后的当前时间
- `wait_until(target_time)` → 精准等待到目标时间

### 3. 抢票执行器 (`core/grabber.py`)

**职责**：执行抢票的两个核心动作。

**Step 1 — 点击"立即购买"**：
- 等待 NTP 定时器触发（开票时间到达）
- 高频检测购买按钮状态（轮询间隔 50-100ms）
- 按钮选择器策略：优先用按钮文本匹配（`text="立即购买"` / `text="立即预订"`），备用 CSS 选择器
- 检测到按钮可点击 → 立即执行 `click()`
- 记录点击时间和耗时

**Step 2 — 点击"提交订单"**：
- 等待页面导航到订单确认页（URL 变化或新 DOM 出现）
- 检测"提交订单"按钮 → 点击
- 等待最多 5s，超时视为失败

**重试机制**：
- 可配置最大重试次数（默认 3 次）
- 可配置重试间隔（默认 500ms）
- 失败场景：按钮未变为可点击、点击后页面未跳转、订单提交超时
- 库存不足/已售罄时停止重试并通知用户

**状态上报**：通过 Qt Signal 向 GUI 发送实时状态：
- `status_changed(str)` — 当前阶段描述
- `log_message(str)` — 详细日志
- `grab_finished(bool, str)` — 抢票完成，成功/失败及原因

### 4. GUI 主窗口 (`gui/main_window.py`)

**布局**：单窗口，上下两个区域。

**控制面板（上方）**：
- "启动浏览器"按钮 — 一键启动带调试端口的 Chrome
- 浏览器连接状态指示（已连接/未连接）
- 开票时间输入 — QDateTimeEdit 控件，精确到秒
- 倒计时显示 — 大字体 Label，格式 `HH:MM:SS.mmm`
- "开始抢票" / "停止" 按钮

**日志面板（下方）**：
- QTextEdit 只读滚动区域
- 实时显示：NTP 校时结果、倒计时状态、按钮检测、点击动作、结果通知
- 每条日志带时间戳（精确到毫秒）

### 5. 线程模型

```
主线程 (PyQt6 事件循环)
  │
  ├─ QThread: AsyncWorker
  │    └─ asyncio 事件循环
  │         └─ Playwright 异步操作 + NTP + 抢票执行
  │
  └─ Signal/Slot 通信
       AsyncWorker → 主线程 (日志、状态、结果)
       主线程 → AsyncWorker (开始、停止指令)
```

Playwright 的异步 API 运行在独立 QThread 的 asyncio 事件循环中，通过 Signal/Slot 与 GUI 主线程通信，互不阻塞。

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| Chrome 未启动/端口不可用 | GUI 提示用户点击"启动浏览器"或检查端口占用 |
| CDP 连接失败 | 重试连接 3 次，间隔 1s，仍失败则提示用户 |
| 未找到大麦网页面 | 提示用户在浏览器中打开大麦网演出页面 |
| 购买按钮不可点击（未开票） | 持续轮询等待，间隔 50-100ms |
| 点击购买后页面未跳转 | 等待 2s 后重试点击，最多重试 N 次 |
| 订单确认页加载超时 | 等待最多 5s，超时后重试整个流程 |
| 库存不足/已售罄 | 记录日志，通知用户，停止重试 |
| 浏览器连接中断 | 尝试重连，失败则通知用户 |
| NTP 校时失败 | 尝试备用 NTP 服务器，全部失败则使用本地时间并警告 |
